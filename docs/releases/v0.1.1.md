# v0.1.1: Control Plane + Starter Auth + Better DX

**Date:** 2026-02-01

v0.1.1 expands Tuvan into a more complete CLI/server workhorse by adding an operational control plane, starter auth wiring, and more automation-friendly diagnostics.

## Highlights

- **Control plane (ops endpoints)**: A separate control plane server (default `127.0.0.1:8081/control`) supports operational workflows like signal injection and config reload.
- **Safe-by-default auth**: Control plane auth hardens automatically when bound to non-loopback hosts; the data plane includes optional starter auth modes (basic auth or bearer token).
- **Better diagnostics for automation**: `tuvan doctor --json` and `tuvan envinfo --json` support machine-readable output for CI and scripts.
- **Updated foundation**: Upgraded to `@fulmenhq/tsfulmen` v0.2.7 and refreshed dependency baselines.

## Quick Start (CDRL)

```bash
git clone https://github.com/fulmenhq/forge-workhorse-tuvan.git my-app
cd my-app
make bootstrap
make run
```

## Notes For Template Consumers

- This repo is meant to be CDRL'd (Clone -> Degit -> Refit -> Launch); downstream projects should customize `.fulmen/app.yaml` early.
- Release signing scripts are included for teams that want verifiable release artifacts, but most template consumers will replace identity, names, and distribution model.
